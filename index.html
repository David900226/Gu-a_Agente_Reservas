Fase 1 — Descargar e importar el flujo base

01 Entrar a Skool y descargar el flujo padre

Entra a la comunidad premium en Skool. Una vez adentro:

  • Ve a Classroom → módulo Agente Reservas
  • Busca la sección Plantilla Base
  • En Resources descarga el archivo llamado "Flujo padre – Reservas MCP"
💡 Tip: Guarda el archivo en un lugar fácil de encontrar, como el Escritorio, para importarlo en el siguiente paso.
02 Importar y activar el flujo en n8n

Ahora importamos el flujo descargado a n8n:

  • Abre n8n y crea un Workflow nuevo (lienzo vacío)
  • Importa el archivo que descargaste (botón Import from file)
  • Una vez abierto, haz clic en Save para guardarlo
  • Actívalo: cambia el estado de Inactivo → Activo
⚠️ Importante: Si no activas el flujo, el agente no funcionará cuando reciba llamadas.

Fase 2 — Crear el asistente en VAPI

03 Registrarse y crear el asistente base en VAPI

VAPI es la plataforma que le da voz a tu agente. Si no tienes cuenta, regístrate primero.

  • Entra a VAPI y ve al menú izquierdo → Assistant
  • Haz clic en Create Assistant
  • En Assistant Name escribe: Agente recepcionista o Agente de reservas
  • Selecciona Blank Template y haz clic en Create Assistant
04 Generar el First Message y System Prompt

Estos textos definen el saludo inicial y la personalidad del agente. Los generamos con un flujo especial en n8n.

  • Ve a Skool → módulo Generador First System Prompt → Resources → descarga el flujo
  • En n8n, crea un workflow nuevo e importa ese archivo
  • Haz clic en Execute Workflow: aparecerá un formulario
  • Llena los datos: nombre del agente, negocio, tipo, ciudad, país, zona horaria
  • Zona horaria Colombia: America/Bogota (búscala en Skool → Zonas Horarias – Guía Rápida)
  • Escribe el horario en palabras: ej. "Atendemos de lunes a viernes de ocho de la mañana a seis de la tarde"
  • Lista los servicios separados por comas
  • Escribe los profesionales en texto claro: ej. "doctor Juan Lozano, doctora Luisa Murillo"
  • Haz clic en Submit y espera a que los nodos se pongan en verde
  • Abre el nodo Datos con doble clic y descarga los dos archivos .txt: First Message y System Prompt
💡 Tip: Si no sabes tu zona horaria exacta, entra a Skool → "Zonas Horarias – Guía Rápida" y cópiala tal cual.
05 Pegar el First Message y System Prompt en VAPI

Ahora cargamos los textos generados en el asistente de VAPI:

  • En VAPI, abre tu asistente → haz clic en Query Assistant
  • Ve a la sección Model en el panel superior
  • Abre el archivo First Message.txt, copia el contenido y pégalo en el campo First Message
  • Abre el archivo System Prompt.txt, copia el contenido y pégalo en System Prompt
  • Arriba, haz clic en Published → selecciona Skip and Publish

Fase 3 — Configurar Voz y Transcripción

06 Configurar la voz del agente

Le vamos a dar una voz en español a tu agente:

  • En VAPI, selecciona la pestaña Voice en el menú superior
  • En Voice Configuration → Provider: selecciona ElevenLabs
  • Activa la opción Add Voice ID manually
  • Abre una pestaña nueva → ve directamente a ElevenLabs (elevenlabs.io) → entra a la página oficial
  • En el menú izquierdo → Voces → filtra por idioma → Spanish
  • Encuentra una voz que te guste → haz clic en los tres puntos → Copiar ID de voz
  • Regresa a VAPI y pega el ID en Custom Voice ID
💡 Tip: Escucha varias voces antes de elegir. La voz de tu agente es lo primero que escuchará tu cliente.
07 Configurar el Transcriber (reconocimiento de voz)

El transcriber convierte lo que dice el cliente en texto. Configúralo así:

  • Ve a Transcriber (al lado de Voice)
  • Provider: Deepgram
  • Model: Nova 3
  • Language: Spanish
  • Activa Background Noising Enable
  • Confidence Threshold: 0.4
  • Activa Use numerals (mejora captura de teléfonos, horas, números)
💡 Tip: "Use numerals" es clave para que el agente capture bien los teléfonos de los clientes.
08 Agregar KeyTerms y configurar el FallBack Transcriber

Los KeyTerms ayudan al agente a entender correos electrónicos correctamente:

  • Ve a Skool → módulo KeyTerms → copia la lista completa (incluye @, ., dominios, etc.)
  • Regresa a VAPI → pega la lista en la sección KeyTerms
  • Baja a FallBack Transcriber:
  •   • Provider: Google
  •   • Language: Spanish
  •   • Model: Gemini 2.0 Flash
  • Haz clic en Published → Skip and Publish

Fase 4 — Conectar VAPI con n8n (Herramienta MCP)

09 Crear la herramienta MCP en VAPI

Esta herramienta es el puente entre VAPI y n8n. Sin ella, el agente no puede ejecutar acciones.

  • En VAPI, ve a Tools en el menú izquierdo → Create Tool → selecciona MCP
  • Ve a Skool → módulo Tool MCP – VAPI → copia el Tool Name: n8n → pégalo en VAPI
  • Copia la Description de Skool → pégala en VAPI en el campo Description
10 Conectar la URL de n8n y agregar la API Key

Ahora le decimos a VAPI a qué URL debe llamar en n8n:

  • En n8n, abre el flujo padre → doble clic en el nodo Agente recepcionista
  • Selecciona Production URL → copia la URL (empieza por https://)
  • En VAPI → Server Settings → pega en el campo Server URL
  • Haz clic en Add Header → en HTTP Headers escribe: authorization
  • En n8n → entra al menú izquierdo → ícono de engranaje → n8n API
  • Crea una API Key nueva (ponle nombre, elige expiración) → cópiala
  • En VAPI, en el campo del valor del header, escribe: Bearer (con espacio) + pega la API Key
  • En MCP Settings selecciona Server-Sent Events
  • Haz clic en Save
⚠️ Importante: Recuerda el espacio entre "Bearer" y la API Key. Sin ese espacio, la conexión fallará.
11 Asignar la herramienta al asistente y configurar End Call
  • En VAPI → menú → Assistant → abre tu asistente
  • Ve a Tools en el menú superior → despliega la lista → selecciona n8n
  • Haz clic en Publish → Skip and Publish
  • En Predefined Functions activa Enable End Call Function (para que el agente cuelgue cuando el cliente termina)

Fase 5 — Ajustes Avanzados del Agente

12 Configurar Advanced: Start, Stop Speaking y Messaging

Estos ajustes controlan cómo habla y escucha el agente durante la llamada:

  • Ve a Advanced en el menú superior
  • Start Speaking Plan: Wait Seconds → 0.7 | Smart Endpoint → VAPI
  • Stop Speaking Plan: Number of Words → 5 | Voice Seconds → 0.4 | Backoff Seconds → 2
  • Messaging → Client Message: activa solo Tool Calls (desactiva todo lo demás)
  • Messaging → Server Message: activa solo Tool Calls
  • End Call Message: escribe un cierre humano, ej: "Fue un placer atenderte. Que tengas un feliz día."
  • Haz clic en Publish → Skip and Publish
💡 Tip: Un cierre cálido y natural mejora mucho la experiencia de tus clientes.

Fase 6 — Flujos hijos y Google Sheets

13 Descargar e importar el flujo hijo "Buscar cliente"
  • Ve a Skool → módulo Flujos hijos → Resources → descarga Buscar cliente
  • En n8n → crea un workflow nuevo → impórtalo con Import from file
14 Crear la base de datos CRM en Google Sheets

Esta hoja de cálculo será la base de datos donde se guardan los clientes y citas:

  • Ve a Skool → módulo Plantilla Google Sheets → descarga el archivo CRM
  • Abre Google Sheets → Archivo → Abrir → pestaña Subir → selecciona el archivo CRM descargado
  • El documento se llama CRM y tiene dos pestañas:
    • 📋 Clientes: columnas email, nombre, teléfono
    • 📅 Registro de citas: guarda agendamientos, reprogramaciones y cancelaciones
15 Conectar Google Sheets con n8n

Ahora conectamos el nodo de Google Sheets en n8n con tu hoja CRM:

  • En n8n, abre el flujo Buscar cliente → doble clic en el nodo Google Sheets
  • En Credentials to connect with → Create new credentials
  • Confirma que esté en OAuth2 → haz clic en Sign in with Google
  • Selecciona tu correo → acepta los permisos → haz clic en Continuar → cierra la ventana cuando diga "conexión exitosa"
  • En Document: selecciona el documento CRM
  • En Sheet: selecciona la pestaña Clientes
✅ Credencial conectada. En el siguiente paso configuramos el filtro de búsqueda para que funcione correctamente.

Fase 8 — Flujo hijo: Verificar disponibilidad

20 Descargar e importar el flujo "Verificar disponibilidad"
  • Ve a Skool → módulo Flujos hijos → Resources → descarga Verificar disponibilidad
  • En n8n → crea un workflow nuevo → impórtalo con Import from file

En este flujo vas a configurar tres nodos: Resolver profesional, Rango horario y Obtener eventos.

21 Crear los calendarios de Google para cada profesional

Cada profesional necesita su propio calendario en Google Calendar. Así el agente puede revisar disponibilidad por separado.

  • Abre Google Calendar con la cuenta de Gmail que usarás para gestionar las citas
  • Arriba a la derecha → ícono de engranaje ⚙️ → Configuración
  • En el menú izquierdo → Agregar calendario → Crear calendario
  • Escribe el nombre del profesional, ej: Abogado David Soler → haz clic en Crear calendario
  • El calendario aparecerá en el menú izquierdo bajo Configuración de mis calendarios → haz clic en él
  • Baja hasta la sección Integrar calendario
  • Copia el ID del calendario (texto largo que termina en @group.calendar.google.com)
💡 Tip: Repite este proceso por cada profesional de tu negocio. Cada uno necesita su propio calendario e ID.
22 Generar el código del nodo "Resolver profesional"

Este nodo es el que conecta cada profesional con su calendario. El código se genera automáticamente con un flujo especial.

  • Ve a Skool → módulo Generador código Resolver profesional → Resources → descarga el flujo Generar code Resolver profesional
  • En n8n → crea un workflow nuevo → impórtalo → haz clic en Execute Workflow
  • Aparecerá el formulario. Llénalo así:
Zona horaria
Tu zona horaria, ej: America/Bogota
Profesionales
Un profesional por línea con este formato:
Nombre Profesional | calendarId
Ejemplo: Abogado David Soler | [email protected]
  • Si tienes más profesionales, agrega cada uno en una línea nueva con el mismo formato
  • Haz clic en Submit y espera que el flujo termine
  • Abre el nodo Convertir código → descarga el archivo código resolver profesional como .txt
  • Ábrelo y copia todo el contenido
💡 Tip: El ID del calendario lo copiaste en el paso anterior desde Google Calendar → Integrar calendario.
23 Configurar los nodos: Resolver profesional, Rango horario y Obtener eventos

Nodo 1 — Resolver profesional:

  • Vuelve al flujo Verificar disponibilidad → abre el nodo Resolver profesional
  • Borra el código que viene por defecto
  • Pega el código que generaste y copiaste en el paso anterior
  • Cierra el nodo

Nodo 2 — Rango horario:

  • Abre el nodo Rango horario
  • Verás dos campos: Time Min y Time Max
  • En el código, busca donde dice setZone → verás "America/Bogota"
  • Reemplaza esa zona horaria por la de tu país en ambos campos (Time Min y Time Max)
  • No cambies nada más → cierra el nodo
💡 Tip: Si no sabes tu zona horaria exacta, ve a Skool → módulo Zonas horarias – Guía rápida y cópiala tal cual.

Nodo 3 — Obtener eventos:

  • Abre el nodo Obtener eventos
  • Ve a Credentials to connect with → Create new credential → Sign in with Google
  • Selecciona tu correo → acepta permisos → cierra la ventana cuando diga "conexión exitosa"
  • No toques nada más en este nodo → ciérralo
  • Haz clic en Save para guardar el flujo Verificar disponibilidad
24 Conectar "Verificar disponibilidad" al flujo padre
  • En el flujo padre → doble clic en la tool Verificar disponibilidad
  • En Workflow asegúrate de que esté en From List → selecciona Verificar disponibilidad (si aparece dos veces, selecciónalo de nuevo)
  • En Workflow Inputs aparecerán tres parámetros: Profesional, Fecha y Hora
  • Haz clic en las tres estrellas ✨ en cada uno de los tres campos
  • Los tres deben quedar con: Defined automatically by the model
✅ Listo: Flujo hijo Verificar disponibilidad conectado. Ahora sigue con Crear cita.
